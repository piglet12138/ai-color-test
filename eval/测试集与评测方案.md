# AI 个人色彩测试 —— 测试集与评测方案

- 项目：`color-style-demo` ｜ 被测接口：`POST https://sg.yaoyuheng2001.me/colorstyle/api/coloranalysis`
- 目标：先解决**测试集有效性**（面向亚洲人的高质量多样样本），再解决**打标/ground-truth 方法学**，最后据此给出**改进的评测指标体系与算法优化方向**。
- 日期：2026-07-02 ｜ 依据：已有 `eval/`（FairFace 34 张 + skin_lab + results_v2）+ WebSearch 调研 + 本地离线验证。

> **根本前提（诚实声明，贯穿全文）**：**个人色彩季型没有客观真值（no ground truth）**。它是主观专家判断，同一个人在不同分析师、不同光照、不同显示器下会得到**不同**结论——业内已知在线测试跨平台约 30% 会拿到不一致结果，线下 draping 专家也会对同一客户判出「暖 vs 冷」相反结论（来源见文末）。因此本方案**不追求「12 季型准确率」这种不存在的指标**，而是转向：①可自动化的**客观子维度一致性**；②**多专家共识**做软标签；③**无标签也成立**的鲁棒性/自一致/公平性指标。

### 推荐组合（分层，务实）
1. **主力回归集 = FairFace 亚洲子集扩样**：继续用 datasets-server API 免下载，把 **East Asian + Southeast Asian 拉到各 40–60 张**，配 South Asian(Indian) 做南亚覆盖。许可 CC BY 4.0 最干净、自带标签、可长期做 CI 分布回归。**这是唯一能兼顾「可商用 + 有属性标签 + 族裔均衡」的公开集。**
2. **画质对照集 = FFHQ 亚洲子样（小规模、离线用）**：手工/用现有人脸属性模型过滤出 ~30 张亚洲高清生活照，仅用于**「画质好时算法是否更稳」**的对照实验（非商用许可 → 只在内部评测，不对外发布图）。
3. **真实产品分布集 = 自建众包（长期最有价值）**：产品实际是用户**手机自拍**，公开集都不完全匹配。建议后续做一个**知情同意 + 自愿授权**的小规模内部众包（50–100 张真实用户自拍，带自报肤色/是否化妆/光照），这是唯一真正代表产品输入分布的样本。**合成脸只用于补极端稀缺格子（如很深肤+冷调），且打标记，不计入肤色/冷暖真值统计。**

## 二、「高质量图片从哪来」现实路径 + 合成脸利弊

- **公开数据集**：优点=零成本、有标签(FairFace)、可复现；缺点=光照野采质量参差、亚洲仍非主体、明星集带妆污染。→ **做基线回归可以，做「产品真实分布」不够。**
- **自建/众包**：优点=完全匹配产品输入（手机自拍、自然光、少妆）、可拿到自报真值（自报冷暖/肤色/是否化妆）；缺点=需知情同意/隐私合规、成本与周期。→ **长期最该投入的一条路。**
- **合成脸（gpt-image-2 / SD / GAN）**：
  - **能用**：补**稀缺分层格子**（例：很深肤×冷调 —— FairFace 里几乎没有）、做**受控扰动测试**（同一张脸只改光照/白平衡，测鲁棒性）。
  - **不能用作肤色/冷暖真值**：生成模型**自身对肤色有系统偏置**（研究表明 SD/GAN 会放大训练集肤色偏差、把肤色同质化、偏亮偏白）。**用合成脸当「客观肤色真值」= 用一个有偏的黑箱去校准另一个黑箱，会污染冷暖/明度评测。**
  - **纪律**：合成人像**一律不入 git**；合成样本在评测里**单独打 `synthetic=true` 标记**，只进鲁棒性/覆盖度实验，**不进客观肤色相关性/公平性统计**。

## 三、可执行的取样与预处理流程（含 QC 门）

目标样本量与分层（**分层是关键，不是总数越大越好**）：

| 分层轴 | 分桶 | 目标 |
|---|---|---|
| 族裔（产品重心=亚洲） | 东亚 / 东南亚 / 南亚 各 ≥40；欧/中东/拉丁/黑 各 ~15（对照） | 亚洲占比 ≥60% |
| 性别 | 男/女 各半 | 平衡 |
| 肤色深浅（实测 L*/ITA 分桶） | 浅(L*>55) / 中(40–55) / 深(<40) 三桶均要有量 | 每族裔覆盖三桶，尤其**深肤要够** |
| 光照质量（QC 门给分） | 通过 QC 的「合格自拍」为主，另留一组「差光照」做鲁棒性对照 | 主集尽量高质量 |

**流程（可复现）**：
1. **取样**：FairFace val 跨多个 offset 拉候选池 → 按 race 过滤亚洲三类 → 每族裔按性别交替、按**本地实测 L\* 分桶**均衡取样（避免只取到中等肤色）。
2. **过 QC 门（复用产品已有客户端质量门）**：曝光/清晰度/削波(clipping)/分辨率 + A4 白平衡 + WB 置信度。**淘汰**过曝/欠曝、糊、高光削波、分辨率过低的图。记录每张 QC 分与淘汰原因（→ 后面可分析「QC 通过 vs 不通过」组算法稳定性差异，验证 QC 门的价值）。
3. **本地客观量化（纯 PIL，无 numpy）**：沿用 `skin_lab.py` 的 YCbCr 肤色掩膜 + 中位像素 → CIELAB，产出每张 **L\*, a\*, b\***。**在此之上补两个派生量**（本次已验证有效，见下）：
   - **Lab 肤色 hue 角** `H = atan2(b*, a*)`（度）——冷暖代理，**比单用 b\* 更准**。
   - **ITA** `= atan2(L*−50, b*)·180/π`——深浅/明度代理（业界标准）。
4. **存档**：`manifest.json`(标签) + `skin_lab.json`(客观量) + QC 分，**只存客观量与派生指标，人像图不进 git 长期库**（版权/隐私）。

## 四、打标 / ground-truth 策略（方法学核心）

**结论先行：季型整体无真值 → 拆成「可客观化的子维度」+「必须人工的整体判断（多专家软标签）」+「无标签也成立的稳定性指标」三层，混合使用。**

### 层 A —— 客观子维度代理真值（可自动、零人工、零 API 花费）
个人色彩三/四轴中，有几轴能用**实测肤色物理量**当**代理真值**来评模型（不是绝对真值，但方向可辩护、可复算）：

| 模型输出轴 | 代理客观量 | 依据 | 本次离线实测相关性(n=34, v2) |
|---|---|---|---|
| **undertone 冷暖** | 肤色 **Lab hue 角 H=atan2(b\*,a\*)**（其次 b\*） | 冷暖=黄(暖)/粉蓝(冷)方向；研究指出**只用 b\* 会漏 a\* 的粉/橄榄信息**，hue 角同时含 a\*+b\* 更全 | **r(undertone,hue)=0.556** > r(,b\*)=0.53 > r(,a\*)=0.30；分组 hue 均值 cool 5.3° / neutral 34.4° / warm 45.2°，**单调可分** |
| **value 明度/深浅** | **L\*** 或 **ITA** | 明度=亮度轴，L\*/ITA 直接对应 | **r(value,L\*)=0.532**，r(value,ITA)=0.506，方向正确 |
| **contrast 对比（可选）** | **发-肤 ΔL\***（头发区 L\* − 肤区 L\*） | 高对比→冬型判据之一；ΔL 是公认代理 | 未实测（需发区分割，列为后续） |
| **chroma 清浊** | 肤色饱和 **C\*=√(a\*²+b\*²)** 弱代理 | 清浊主观成分高，代理最弱 | 仅作参考，不作主指标 |

→ **这三轴（尤其冷暖、明度）可做成自动化「一致率/相关」指标**，比评 12 季型整体**可评得多、且不花 API**（离线用已有 skin_lab 就能算）。

### 层 B —— 多专家 / 众包共识（软标签，必须人工，有成本）
季型**整体**只能靠人。方案：
- **N=3–5 名（业余亦可）标注者**，对同一批脸各自给「四季 + 冷暖 + 深浅」。
- 计算**标注者间一致性**：四季用 **Fleet/Cohen's Kappa**、连续轴用 **ICC**。**先量分析师们彼此有多不一致**（预期四季 Kappa 中等、12 季型很低）——这本身是诚实呈现「无真值」的证据。
- 取**多数投票**为「软标签（consensus label）」，模型 vs 软标签只报**与人类共识的一致率 + 混淆矩阵**，**明确标注「这是与人类共识的吻合度，不是准确率」**。
- **成本估计**：3 名标注者 × 100 张 × ~20 秒/张 ≈ 每人 ~35 分钟，一轮可控；关键是**招募与口径统一**（先给 draping 判据说明书）。可用低成本内部众包，不必付费专家。

### 层 C —— 无标签也能做的指标（零真值、零人工，最该先做）
即使完全没有标签，以下指标都成立且**信息量大**：
1. **Test-retest 自一致性**：同一张图重复调 2–3 次，看四季/12 型/三轴是否稳（现基线：12型 50% 抖、四季 83%）。**低成本、直接反映随机性问题。**
2. **光照鲁棒性**：对同一张图做**白平衡/亮度/色温增广**（纯 PIL 可做），看结论是否翻转。**这是最贴合「无真值但可辩护」的核心指标**——一个好的色彩算法不该因为室内暖光就把冷调判成暖。
3. **跨肤色公平性**：按 L\*/族裔分层，看**冷调出口率、季型分布**是否对某肤色段系统性缺失（现基线：深肤缺冷调）。
4. **分布健康度**：四季/12型是否坍缩（现基线：秋 61%）——**坍缩即先验污染的信号**。

### 推荐组合
- **每次上线前 CI 必跑（全自动、~0 人工、少量或零 API）**：层 A（客观子维度相关/一致率，离线）+ 层 C（分布 + 公平性，离线用已有结果；鲁棒性用增广小规模调 API）。
- **季度/大改一次（有人工成本）**：层 B（多专家软标签共识）做一次背书，报「与人类共识吻合度 + Kappa」。
- **绝不宣称**「季型准确率 X%」这种伪指标。

## 五、改进的评测指标体系（不只看分布）

| 类别 | 指标 | 怎么算 | 通过线（建议） |
|---|---|---|---|
| **客观一致（层A）** | undertone↔hue 相关、value↔L\*/ITA 相关 | Pearson r + 分组均值单调性 | r≥0.5 且分组单调；**冷暖分组 hue 均值须单调分离** |
| **客观一致（层A）** | 冷暖三分类「符合物理方向」率 | 判 cool 的脸 hue 应显著低于判 warm | cool 组 hue 均值 < warm 组，且无严重逆例 |
| **分布健康** | 四季/12型分布熵、最大类占比 | 计数 → 占比/熵 | 单一四季占比 **<50%**；12型用到 ≥8 类 |
| **公平性** | 分肤色段(浅/中/深)、分族裔的**冷调出口率差**、季型分布差 | 分层计数 + 差值 | 深肤段冷调率 **>0**；各段季型分布不应互斥坍缩 |
| **混淆/共识（层B）** | 与人类共识的四季混淆矩阵 + Kappa | 多数投票软标签 | 报告值 + 明确「非准确率」 |
| **自一致（层C）** | test-retest 一致率（四季/12型/三轴） | 同图重复 N 次众数一致比例 | 四季 ≥90%、三轴 ≥85%（12型不设硬线，抖动是本质） |
| **鲁棒性（层C）** | 光照增广翻转率 | 白平衡/亮度扰动后结论改变比例 | 冷暖翻转率 **<15%** |
| **校准** | 若模型给置信度：可靠性曲线(ECE) | 分箱预测置信 vs 实际一致率 | 低置信样本应更易抖动（单调） |
| **工程** | API 成功率、延迟、429 率、解析失败率 | 计数 | 成功率 100%、解析失败 0 |

## 六、算法优化建议（结合已有 v2 结果）

**v2 已取得的进展（本次离线复算确认，n=34）**：相比 v1，**冷调 4→7 / 34**、**秋 85%→61%**、**Black 冷调 0→1**、**12 型用到 7→8 类**、性别检测 33/34，`undertone/value/chroma` 三轴字段已产出可审计。**方向对，但仍偏暖偏秋。** 进一步优化：

1. **冷暖用 hue 角而非 b\*——给模型/后处理更好的判据**：本次实测 **hue(atan2(b\*,a\*)) 比 b\* 更能分冷暖**（r 0.556 vs 0.53，且 cool/neutral/warm 均值 5.3/34.4/45.2 单调分离）。可在 prompt 里明确「冷=偏粉蓝、暖=偏黄橙，olive/粉调也是冷信号」，并考虑**服务端做一次本地 hue 估计作为软先验/校验**（判 warm 但实测 hue 很低 → 触发复核）。
2. **白平衡/光照归一预处理**：产品输入是手机自拍，室内暖光是「默认判暖」的主因。**在送模型前做一次白平衡归一**（已有客户端 A4 白平衡 + WB 置信度可用）；WB 置信度低时降权冷暖结论或提示补拍。
3. **多次投票降随机**：test-retest 12 型仅 50% 稳 → 同图 **2–3 次取众数**（或降温/固定 seed）返回，四季/三轴会明显更稳；成本换稳定，对边界季型尤其值。
4. **为深肤保冷调出口**：深肤(L\*<40)16 张仅 3 冷调。prompt 显式给「深肤也可为深冬/冷冬」的判据与示例，避免「深→暖→秋」链式坍缩。
5. **返回数值化三轴 + 置信度**：当前 undertone 只有文本档。让模型输出**冷暖/明度/清浊的数值分 + 置信度**，既利于上面的校准指标，也利于「低置信触发复拍/复核」的产品逻辑。
6. **打破「默认判秋」先验**：分布仍秋 61%。prompt 继续强制**先独立判三轴再合成季型**，并对春/夏/冬给足正例，必要时对四季做轻度**分布正则提示**（"不要默认判秋"）。

## 七、可选的小规模验证结果（本次离线完成，**0 次新 API 调用**）

全部基于已有 `results_v2.jsonl` + `skin_lab.json`，**未花任何反代额度**：

- **客观一致（层A）**：
  - `r(undertone, hue)=0.556`，`r(undertone, b*)=0.53`，`r(undertone, a*)=0.30` → **hue 角是更好的冷暖代理**；分组 hue 均值 cool **5.3°** / neutral **34.4°** / warm **45.2°**（单调可分）。
  - `r(value, L*)=0.532`，`r(value, ITA)=0.506` → 明度方向正确，L\* 与 ITA 相当。
  - 冷暖分组 b\* 均值：cool 7.3 / neutral 13.0 / warm 19.6（单调）。
- **分布（v2）**：四季 秋 21 / 冬 6 / 春 5 / 夏 2 → **秋 61%**（v1 为 85%）；12 型用到 **8 类**；undertone warm 23 / cool 7 / neutral 4。
- **公平性**：冷调 **7/34**（v1 4/34）；Black 冷调 **1/4**（v1 0）；深肤(L\*<40) 冷调 3/16——**改善但仍偏少**。
- **性别检测**：33/34。

→ 结论：**v2 纠偏方向正确且量化可见改善**，但「偏暖偏秋 + 深肤缺冷调」未根除；**hue 角**被证实为更优冷暖客观代理，建议纳入指标体系与算法。

---

## 八、诚实局限

1. **季型无客观真值**——本方案所有「一致率」均为「与客观子维度代理量」或「与人类共识」的吻合度，**不是准确率**；层 A 的代理量（hue/L\*/ITA）也只是方向可辩护的近似，非绝对真值。
2. **本地肤色量为粗略近似**：YCbCr 掩膜 + 中位像素、无关键点定位、图未白平衡校准；研究指出 **ITA/b\* 对蓝通道噪声敏感**，未标定管线需谨慎（本方案因此只做方向性/相关性判断，不做绝对分类）。
3. **样本仍小**：现 34 张，冬/春/深肤格子稀疏，分族裔仅趋势不具显著性——**扩样到亚洲各 40+ 是前置动作**。
4. **合成脸不可作肤色真值**：生成模型自带肤色偏置，只用于补格子/鲁棒性，且标记隔离。
5. **FairFace 光照/画质仍是野采**：与产品真实自拍分布有差；**众包真实自拍集**才是长期真相来源，但需隐私合规。
6. **多专家共识受招募与口径影响**：Kappa 预期不高，正说明「无真值」，不应据此反推模型错。
