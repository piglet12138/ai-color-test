# StyleLab：拍一张照片，让 AI 读懂你的色彩与形象

> 一个用 GPT-5 视觉 + gpt-image 图像编辑搭起来的「AI 私人形象顾问」——上传一次照片，就能测色、配穿搭、试花纹/发色/发型/妆容/首饰，并且所有效果图都直接生成在**你本人**脸上。
>
> 线上体验：<https://sg.yaoyuheng2001.me/colorstyle/app.html>
> 源码：<https://github.com/piglet12138/ai-color-test>

这篇文章分成两部分：

- **第一部分 · App 流程体验与讲解**——从用户视角走一遍，它到底能做什么、每一步长什么样。
- **第二部分 · 技术实现介绍**——背后的色彩理论、调用的 API、"保脸"换装的一整套设计，以及我用公开人脸数据集做的评测实验。

# 第一部分 · App 流程体验与讲解

StyleLab 被设计成一个**手机 App 的样子**（404px 宽的 phone frame + 底部 5 个 Tab + 引导流），但它其实是一个纯网页，打开链接就能用。整个产品的核心理念只有一句话：

> **上传一次照片 → AI 建一份"形象档案" → 之后所有测试都复用这一张照片，直接把效果生成在你身上。**

## 1. 欢迎 & 建档：只上传一次

第一次进来是引导流：欢迎页 → 上传页 → 分析中 → 色彩报告。

上传页不是简单丢个文件选择框，而是带了一整套**拍照引导**，因为测色对照片质量非常敏感：

- 📸 面朝**自然光**（窗边最佳），避免逆光、顶光、暖黄灯
- **素颜或淡妆**，关闭美颜/滤镜，用**原图**别用截图
- 正脸、露额头、表情放松
- 有条件的话，在下巴旁举一张 **A4 白纸**——App 会自动用它做白平衡校色

你可以「📷 拍照」（App 内置全屏相机，带人脸椭圆引导框）或「从相册上传」。选完图立刻显示预览，并跑一遍**客户端质量门（QC）**：分辨率够不够、曝光合不合适、糊不糊、有没有过曝削波——不合格会给出**具体的重拍建议**（"太暗了，找更亮的自然光""拿稳手机等对焦"），而不是拍完传上去才发现测不准。

同时可以填一点基础信息（性别 / 体型 / 身高 / 体重，都可以留空让 AI 判断），以及一个**默认关闭**的匿名数据贡献勾选（后面会讲）。

## 2. 色彩报告：你的"形象档案"

点「AI 测色，生成档案」，进入一个有轨道动画的分析屏，几秒到十几秒后出报告：

- **季型判定**：四季十二型体系里的一个（如"柔秋 Soft Autumn"），带一个**真实的置信度**（不是固定 80%，照片质量差会自动降）
- **个人色板**：推荐色 / 中性色 / 避免色，每个都有中文名 + 准确色号
- **妆容色**：眼影 / 腮红 / 唇 / 眉 的具体色调建议
- **一段通俗的解释**：像顾问当面讲解一样告诉你"为什么是这个季型"——**故意不出现 Lab、色号、ΔE 这些技术黑话**，而是用"偏暖的米杏色""深棕近黑的发色""明暗对比适中"这种自然说法

如果你登录了，这份档案会存进服务器，下次进来**直接免重传**，回到首页就能看到它。

## 3. 首页：四个入口，共用一张照片

建档完成进入 App 首页。顶部是你的档案摘要，下面是四张功能卡：

| 入口 | 做什么 | 大概耗时 |
|---|---|---|
| 🎨 **色彩测试** | 生成一张小红书风的"个人色彩分析"海报，推荐/一般/避免色都穿在你身上 | ~1 分钟 |
| 👗 **穿搭试穿** | 出 2-3 套成套造型，每套配色 + **真人上身预览** + 造型点评 | ~2 分钟 |
| 💇 **形象测试** | 花纹 / 发色 / 发型 / 妆容 / 首饰，看每种变化在你身上的样子 | 每项 ~1 分钟 |
| 🗂️ **我的历史** | 过往所有结果，按类型汇总、可点开看详情 | — |

关键在于：**这四个入口全都复用建档时那一张照片**，你不需要每次重传。

## 4. 色彩测试海报

用你档案里的照片，生成一张色彩分析大图。做法很巧妙——**不是让 AI 一次画出整张海报**（那样文字会糊、脸会不一致、颜色也不准），而是：

1. GPT-5 看照片出完整的色彩 + 妆容 JSON（准）
2. 并行生成 3 条"色带"：同一个你、并排穿 6 种不同颜色的针织衫（脸一致、颜色按序、无文字）
3. 前端把色带 + 标签 + 妆容 + 肤色拼成海报，一键导出 PNG 下载

## 5. 穿搭试穿：把衣服"穿"到你身上

这是最出效果的部分。AI 根据你的季型和色板给 2-3 套成套造型，每套有配色方案、单品描述、一段有质感的中文点评，然后——**把这套穿搭直接生成在你本人的照片上**。

这里有两种模式：

- **纯换装**：严格保留你的脸、发型、体型、背景，只换身上的衣服；
- **氛围感模式（默认开）**：不只是换衣服，而是把你放进一张**贴合季型的氛围大片**里——柔秋会给你咖啡馆的黄金暖光、慵懒碎发、松弛表情、暖棕调；冷冬会给冷调高对比、利落线条。身份保持得很好，效果远胜纯灰底证件照。

## 6. 形象测试实验室

一次上传，五种测试：**花纹 / 发色 / 发型 / 妆容 / 首饰**。点任意一项，AI 先推荐适合你的若干选项（带理由），再生成一张"同一个你、并排展示各选项"的对比图。发色、发型、首饰都能保持是同一张脸。

妆容和首饰会**按性别给不同方案**：男士给清爽素颜感 / 控油雾面 / 男士手表戒指项链，女士给氧气妆 / 韩系水光 / 耳饰项链等——不再"默认按女性处理"。

## 7. 我的：历史 + 数据撤回

「我的」Tab 里，所有结果按测试类型汇总（可筛选、可点开看详情、可删除），历史图片长期保存。如果你当初勾选了匿名数据贡献，这里能看到已贡献数量，并**一键撤回删除**。

# 第二部分 · 技术实现介绍

整个后端是**一个零依赖的 Node 文件**（`server.mjs`，纯 `node:http` + JSON 存盘，不跑 `npm install`），前端是**原生 JS**。所有 AI 能力都走**同一个 OpenAI 兼容反代**：GPT-5（多模态，看图 + 推理）负责所有"判断/文字"，gpt-image 负责所有"生图/换装"。

## 2.1 色彩理论基础：四季十二型

产品的理论内核是**季节性色彩分析（Seasonal Color Analysis）**中的 **12 季型体系**。它把人按三个正交维度分类：

- **冷暖（Undertone）**：cool / warm / neutral——看肤色、唇色、血管泛粉泛蓝（冷）还是泛黄泛金（暖）；
- **明度（Value）**：浅 / 中 / 深；
- **纯度（Chroma）**：清透鲜艳 / 柔和浑浊。

三轴组合落到春/夏/秋/冬四大族、每族三型共 12 个季型（浅春、暖春、亮春、浅夏、冷夏、柔夏、柔秋、暖秋、深秋、亮冬、冷冬、深冬）。每个季型对应一套"显气色"的色板和一套"显疲惫"的避免色。

我把这套体系固化成一份**参考数据库**（`presets.mjs`）：12 季型每个都带 `tone/value/chroma` 三轴标注 + 6 个代表色（中文名 + 精确 hex）+ 避免色，另外还有各测试项的扩展选项目录（花纹 15 种、发色 12 种、发型男女各十几种、妆容/首饰分性别）。这份 KB 会被压缩成紧凑字符串**注入到 prompt 里**，让 GPT-5 的分类必须从 12 季型里选、推荐色优先采用标准代表色——保证输出落在专业体系内、色号一致、不乱飘。

## 2.2 客户端 CV 预处理：先量再问

在把照片发给大模型之前，浏览器端先做一遍轻量 CV（`public/color.js`，纯 Canvas 无依赖）：

- **肤色提取**：用 YCbCr 经验阈值做皮肤掩膜，取中部区域皮肤像素的中位 RGB；
- **发色提取**：取画面上部、中间列里最暗的一撮像素均值；
- **CIELAB 转换**：把肤/发色转成 CIELAB，拿到 `L*`（明度）、`a*`/`b*`（冷暖代理），并算发肤明度对比 `ΔL`；
- **整体色板**：一个极简 k-means 抽出 6 个主色。

这些客观测量值会连同照片一起喂给老的混合式端点（`/api/diagnose`）。**注意一个架构上的取舍**：旗舰的 `/api/coloranalysis` 是**纯视觉**的——只发图片让 GPT-5 判季型，不喂客户端算的 Lab 参数。为什么？因为单张非受控自拍的绝对色值受光照影响极大，喂进去反而可能误导；客户端 CV 更多用在**质量门**和**离线评测校验**上（见 2.6）。

## 2.3 调用的 API：一个反代打通所有能力

所有 AI 调用都指向一个 OpenAI 兼容的 Base URL，三个模型角色：

```
CHAT_MODEL  = gpt-5.4-mini   // 看图 + 推理（多模态），所有"判断/文字"
IMAGE_MODEL = gpt-image-2    // 文生图（无照片时兜底）
EDIT_MODEL  = gpt-image-2    // 真人换装（在用户照片上改穿搭）
```

**看图 + 推理**走标准 `/v1/chat/completions`，content 里塞 `{type:"text"}` + `{type:"image_url", image_url:{url:"data:image/...;base64,..."}}`，并用 `response_format:{type:"json_object"}` 强制结构化输出。GPT-5 系列都是多模态的，标准 `image_url` 就能看图；**故意不传 `temperature`/`max_tokens`**，避免撞上推理模型的参数限制。

模型偶尔会在 JSON 外包一层 ``` fence、写行内注释或留尾逗号，所以有一个**宽松解析器 `parseLoose`**：先剥 fence，失败就正则抠出最外层 `{...}`，再去注释、去尾逗号才 `JSON.parse`——这是让 LLM 结构化输出稳定落地的必要保险。

**换装/生图**走 `/v1/images/edits`（multipart）和 `/v1/images/generations`，返回 `b64_json`。gpt-image 很慢（约 100 秒/张），且偶发 5xx，所以做了**指数退避重试**（edits 4 次、超时 200s）。

## 2.4 "保脸"换装：产品成败的关键

把衣服/妆容/发色画到用户本人脸上，最大的风险是**身份漂移**——生成出来不像本人了。这套"保脸"设计是产品体验的命门，做了几层：

**① Prompt 层的硬约束（借鉴社区"保脸配方"）。** 换装 prompt 里**硬性列举「不要改」的清单**并**重复强调关键约束**：

```
【严格保留】人物的脸型、五官比例、表情、发型、发色、肤色、体型体态、
姿势与背景、以及性别；保留真实皮肤纹理、毛孔与细纹，不要磨皮、不要瘦脸、
不要换脸、不要改变人脸/身材/性别。
就是这个人本人！就是这个人本人！只换衣服，其它一切不变。
```

保留皮肤纹理/毛孔这条尤其重要——它防止模型"顺手美颜"导致身份漂移。

**② 用户知识库注入（subjectLine）。** 把档案里的性别/体型/肤色/发型拼成一段"画面人物固定特征"约束串，**每次生图前置注入**：

```
【画面人物固定特征，必须严格保持】性别为男性（务必生成男性，不得改变性别）、
匀称体型、中性偏暖肤色、黑短发。
```

这直接修掉了早期一个严重 bug：coloranalysis 原来不返回性别 → App 默认按"女"处理 → 男用户生成出女图。现在性别由 AI 判定并锁进 subjectLine，色带 prompt 也显式"务必男性/女性、保持性别不变"。

**③ 只用 img2img，不做换脸。** 我评估过 InstantID / PuLID / ComfyUI 自建管线（本机有 RTX 4060），最终**只用 prompt + gpt-image 的 edits**，没上任何 face-swap 模型。氛围感大片也是在**用户本人照片上 img2img**注入季节氛围（`presets-vibe.mjs` 里把四季映射成场景/光线/表情/发型/调色的知识库），而不是把脸换到一张模板大片上——这样保脸更稳、也不需要付费的换脸服务。

**④ 拆任务，别让生图干它不擅长的事。** 一个反复验证的洞察：**别让 gpt-image 一次生成带文字的整张海报**（文字糊、脸不一致、颜色不准）。正确拆法是——**GPT-5 出文字和颜色（准）+ gpt-image 只出色带人像（准）+ 程序排版（准）**。海报里的每条色带都是"同一人并排穿 N 色针织衫"，脸一致、颜色按序、无文字，最后前端用 `html2canvas` 拼版导出。

## 2.5 工程优化

- **图片缓存**：生成的图按"模型+尺寸+prompt+图片哈希"存盘，重复请求秒回（24h TTL 自动清理）；返回相对 url 兼容子路径部署。
- **历史图持久化**：cache 有 24h TTL，所以存历史时把引用的 cache 图复制进 `media/` 长期保存并改写 url。
- **减卡顿**：换装底图从 PNG 改 JPEG 0.9（小很多、编码快、省 sessionStorage）；处理 canvas 上限 1400px 省内存。
- **并行**：色带 / 穿搭预览 / 变体图都是一键三张并行出，把等待压到单张耗时。
- **零依赖 + 相对路径**：`node server.mjs` 无 npm 依赖，前端全用相对路径，所以能干净地挂在 nginx 子路径 `/colorstyle/` 下，并做了 GitHub Actions push→SSH→`git reset --hard`→restart 的自动部署（部署密钥用 forced-command 绑死，即使泄漏也只能触发这一条部署、拿不到 shell）。

## 2.6 评测实验：拿公开人脸数据集测偏差

季型判定有个根本问题——**它没有客观 ground-truth**。业内跨平台判定约 30% 不一致，专家判冷暖甚至会给出相反结论。所以我不追求"准确率"，而是用一个可复现的实验去**测分布、一致性和方向合理性**。

**数据集**：FairFace validation（族裔/肤色多样性最好且自带 race/gender/age 标签），跨 13 个 offset 拉候选池后按族裔分层取 **34 张**（7 族裔均衡、男女交替）。用纯 PIL（无 numpy）在中心区域做 YCbCr 肤色掩膜，取肤色像素中位 RGB → CIELAB，拿 `b*`（暖度代理）和 `L*`（深浅代理）做方向性校验。上游并发上限=1（并发即 429），严格单并发、429 退避重试。

**第一版结果暴露了严重偏态：**

- 四季分布 **85% 判"秋"**，undertone **88% 判"偏暖"**，12 型只用到 7 类且集中在深秋/柔秋；
- 冷暖方向本身是对的——判"偏冷"的样本实测 `b*` 均值 3.4，判"偏暖"的 18.0，Pearson r≈0.50；明度维度更是高度吻合（深秋 L*≈36 最低、浅春 L*≈74 最高）；
- 但存在**"深肤→判暖/判秋"的潜在族裔偏向**：东亚是唯一被合理分出冷调的群体，其余 6 族裔（尤其深肤的 Latino/Black）几乎一律"秋 + 偏暖"，缺冷调出口；
- **12 型 test-retest 一致率只有 50%**（四季大类稳在 83%），细分型在边界处抖动明显。

**针对性纠偏。** 我据此重写了色彩 prompt（`COLOR_SYS`）：强制**先分冷暖/明度/纯度三轴独立判断、再合成季型**，明确要求"警惕室内暖光/滤镜把脸染黄""深肤色同样可能是冷调（→深冬/冷冬），绝不能一律判暖""刻意纠偏无差别判秋的倾向"，并要求输出数值化的三轴字段和一个别都给同一个值的真实置信度。

**第二版回归（同 34 张）验证纠偏生效：**

| 指标 | v1 | v2 |
|---|---|---|
| 判"秋" | 85% | **62%** |
| 判"暖" | 88% | **68%** |
| 冷调样本数 | 4 | **7** |
| Black 出现冷调 | 0/4 | **1/4** |
| 冬 / 春 出口 | 1 / 1 | **6 / 5** |
| 性别检测正确 | — | **33/34** |

偏态明显缓解、冷季和春季有了出口、深肤开始能被判冷调——虽然仍略偏秋，但方向对了。整套评测脚本（`eval/`）可复用，可以作为**上线前的离线分布回归**。

**顺带一个算法层的发现**：离线实测 **Lab 的 hue 角 `atan2(b*, a*)` 比单看 `b*` 更能分冷暖**（cool/neutral/warm 三组 hue 均值 5.3/34.4/45.2 单调分离）——这既是更好的评测指标，也指向一个更好的算法方向：把 coloranalysis 也改成"先算 hue + 白平衡归一再连图带参喂 agent"。

## 2.7 合规的数据采集

评测需要更多真实样本，所以做了一个**明确同意制**的数据管线：建档屏有一个**默认关闭**的 opt-in 勾选 + 隐私说明页；只有勾选（`consent === true`）才会把照片和测色结果匿名入库，用户能在「我的」里看到贡献数并**一键撤回删除**（按 id 或按登录用户删图 + 删元数据）。数据集目录 gitignore、不入库。

## 小结

StyleLab 想证明的事情很简单：**用一个通用视觉大模型 + 一个图像编辑模型，加上一层扎实的领域知识（色彩理论）和工程约束（保脸、拆任务、评测纠偏），就能把一个"看起来需要专业顾问 + 专业修图师"的体验做成一个零依赖的小网页。**

难点从来不在调 API，而在：把主观的色彩体系翻译成可注入的 prompt 约束、把生图任务拆成模型各自擅长的小块、用硬约束死死锁住用户身份、以及用一个可复现的实验诚实地量出模型的偏差再去纠偏。

> ⚠️ 这是个人探索项目，非商用；测色仅供娱乐/参考——季型判定没有客观标准答案，请把它当有趣的起点，而不是权威结论。
